# 小米/红米 澎湃OS 降级 MIUI 刷机教程

> **⚠️ 极度危险警告：刷机有风险，操作需谨慎！**
> 错一步即变砖，小米售后通常需花费约 **1000元更换主板**。
# CVE-2025-21479 — 高通 Adreno GPU 本地提权漏洞

**CVE-2025-21479** 是存在于 **高通骁龙8系列（骁龙8 Gen1/Gen2/Gen3 等）Adreno A7xx GPU 微码（固件）** 中的高危本地提权漏洞,允许软件利用显卡驱动缺陷直接获取手机 Root权限。
## 📝 前置准备

### 1. 核心风险提示
*   **数据全清**：刷机将清除所有照片、视频及应用数据，请务必提前备份。
*   **系统限制**：仅 **安卓15 / 澎湃OS 2.0 或 3.0** 可降级。安卓14/澎湃1.0需先升级系统。
*   **包体匹配**：必须下载视频简介中对应机型的刷机包，严禁混用。

### 2. 电脑环境配置
*   **操作系统**：Windows 10 或 Windows 11。
*   **杀毒软件**：关闭 Windows 安全中心查杀（刷机工具涉及底层寻址，属正常误报）。
*   **数据线**：必须使用原装或高品质 Type-C 数据线。
    *   台式机：需插机箱后部主板 USB 口。
    *   笔记本：需插电源。
    *   严禁使用网吧扩展坞。

### 3. 文件命名规范（防呆措施）
*   工程包 `HyperOS to MIUI` 重命名为：
    > `01_GongCheng必须先刷_对应视频前10分钟步骤`
*   最终 MIUI 包 `机型代码-MIUI-14.x` 重命名为：
    > `02_最后才能刷miui-先去FastbootD页面_再进FASTBOOT刷`

## 🛠️ 操作步骤详解

### 第一阶段：驱动与漏洞准备

**步骤 1：安装驱动**
电脑打开 `{1}驱动{刷机前刷入}` 文件夹，安装全部驱动软件。

**步骤 2：手机连接与授权**
*   手机开启“开发者选项”及“USB调试”，连接电脑并点击“允许授权”。
*   **异常处理**：若检测不到设备，请检查接口接触或更换数据线，最好使用小米原装线。

**步骤 3：漏洞利用**
*   电脑打开 `{3}漏洞` 文件夹，选择与当前系统版本严格对应的 `boot` 文件（例如澎湃3.0.4对应3.0.4的boot）。
*   运行工具，若卡在第6步，等待最多一小时即可。

---

### 第二阶段：ABL 分区破解（高危步骤）

**步骤 4：推送文件**
使用紫罗兰工具箱推送两个文件：
*   推送 `01-GONGCHENG` 工厂包内的 `abl` 文件。
*   推送 `{4}安卓KSU` 安装包。

**步骤 5：获取 Root 权限**
*   手机端打开文件管理，安装 KSU (KernelSU)。
*   进入软件内的“超级用户”列表，搜索 `shell` 并赋予 Root 权限。

**步骤 6：刷入 ABL 命令**
*   紫罗兰工具箱打开 CMD 命令行，**分段**输入以下两条命令（每输入一条按一次回车）：

adb shell su -c dd if=/sdcard/abl.elf of=/dev/block/by-name/abl_a

adb shell su -c dd if=/sdcard/abl.elf of=/dev/block/by-name/abl_b
> **💡 提示**：此步骤是为了将破解版引导文件写入 A/B 分区的 ABL 区域，为后续刷机解锁权限。执行后切勿直接点击“重启系统”，若误触导致红字报错，手动进入 Fastboot 即可。

---

### 第三阶段：工程包刷入

**步骤 7：进入 Fastboot 模式**
*   紫罗兰工具箱重启至“Fastboot”模式。
*   若电脑未检测到设备，请参考此链接排查：<https://champhoon.xyz/note/mi-fastboot-fix/>

**步骤 8：刷入工程包（最关键步骤）**
*   在 Fastboot 模式下，必须选择 `01_GongCheng` 开头的工程包（文件大小通常在 4GB 以下）进行刷入。
*   **🚨 严禁在此步选择 `02` 开头的 MIUI 包！否则将触发防回退机制，导致无法开机且永久丢失刷写权限！**

> **报错处理：**
> *   `fastboot devices console` 报错：重新插拔数据线，手动重进 Fastboot。
> *   `Erase is not allowed in lock State`：步骤 6 未成功，需从头重来。

---

### 第四阶段：切换 FastbootD 模式

**步骤 9：进入 FastbootD 模式**
工程包刷完后，手机应自动进入 FastbootD 模式。

> **异常情况处理：**
> *   **卡在开机 Logo 无限重启**：长按【音量- + 电源键】强制进入 Fastboot，再通过工具箱重启至 FastbootD。
> *   **进入工厂测试系统**：通过工具箱重启至 FastbootD。

> **📌 注意**：此步骤必须经历，不可跳过。对应视频第 10 分 20 秒进度条。

---

### 第五阶段：最终降级 MIUI

**步骤 10：转回普通 Fastboot 模式**
*   确认手机处于 FastbootD 页面后，通过工具箱重启至第二排的 **FASTBOOT**（黄色界面）。
*   未经过 FastbootD 直接刷 MIUI 将导致黑砖且无法自救。

**步骤 11：刷入 MIUI 系统**
*   在普通 Fastboot 模式下，选择 `02_最后刷miui` 的包，点击开始刷入。
*   等待约 15 分钟即可开机，降级成功。
